# Security Policy

## Reporting a Vulnerability

Please do not open a public issue for a vulnerability.

Use GitHub private vulnerability reporting if it is enabled for the repository. If it is not enabled, contact the repository owner through a private channel and include enough detail to reproduce the issue.

## Scope

Security-sensitive areas include:

- command execution through AI tooling
- microphone capture and audio playback
- local model downloads and cache paths
- configuration handling
- session persistence

Avoid sharing credentials, private prompts, local config files, or audio recordings in public reports.
